## Goal

Build the foundational release-pipeline tooling: a bash script that takes a `<version>` arg, verifies repo + package.json + CHANGELOG state, runs install/build/test, stages framework files into `dist-pkg/`, tars them with the right exclude/include logic, and emits the tarball to `/tmp/neato-hive-v<version>.tar.gz` with a sidecar `.checksums.txt` containing the SHA-256.

Plus a small audit script that diffs staged contents against the expected REPLACE_LIST so a human (or B.2's automation) can confirm the tarball composition before pushing to the site repo.

**B.1 ships LOCAL tarball generation only.** It does NOT push to the site repo (`Daniel-Neato/neato-hive-site`) — that's B.2. It does NOT update `current.json` or `index.json` — that's B.2. After B.1, Bob (or owner) can run `bash scripts/release.sh 1.5.0-test` locally and produce a verifiable tarball + checksum sidecar; B.2 wires the upload + site-repo update + Vercel rebuild flow.

## Architectural givens (locked from prior decisions, captured here for grep-ability)

- **Q2 resolution:** tarball EXCLUDES `node_modules/`. Update flow runs `pnpm install --frozen-lockfile` post-extract. ABI-safe across Node 18/20/22.
- **`packageManager` pinning:** `package.json` must declare `"packageManager": "pnpm@<version>"`. Corepack respects this. B.1 verifies the field is present in pre-flight.
- **REPLACE_LIST** (items that overwrite on update — ship in tarball):
  - `dist/` (TypeScript compile output)
  - `bin/` (the `hive` shell wrapper + helpers)
  - `templates/` (agent templates including the new A.1 site skeleton)
  - `shared/` (CRITICAL-RULES.md, CREDENTIALS.md, GLOBAL-TOOLS.md, NEATO-NARRATIVE.md, exchange/)
  - `skills/` (named skills)
  - `dashboard/` (placeholder — only staged if directory exists; future Phase E ships it)
  - `package.json`
  - `pnpm-lock.yaml`
  - `VERSION` (NEW — top-level file containing the version string, B.1 generates as part of staging)
  - ~~`CHECKSUM`~~ — vestigial entry. **B.1 does NOT include CHECKSUM in tarball** (Q4 resolution: in-tarball CHECKSUM is circular — checksum of the tarball can't be inside it before computing). Sidecar `.checksums.txt` is canonical.
- **PRESERVE_LIST** (items NEVER touched by tarball — never in tarball):
  - `agents/`
  - `data/`
  - `config/*.local.yaml`
  - `.env`, `.env.local`, `*.local.*`
  - Plus build/dev artifacts: `node_modules/`, `.git/`, `src/`, `tsconfig.json`, `pnpm-workspace.yaml`, `.local/`, `dist-pkg/` (the staging dir itself)
- **Staging dir:** `<repo>/dist-pkg/` — same FS as repo root (avoids cross-FS copy issues for the `cp -a` operations during staging). Git-ignored. B.1 adds `dist-pkg/` to `.gitignore` if not already present.
- **Output:** `/tmp/neato-hive-v<version>.tar.gz` + `/tmp/neato-hive-v<version>.checksums.txt`. The `/tmp/` location is intentional — output is ephemeral; B.2 picks it up and pushes to the site repo's `releases/v<version>/`. Don't pollute the framework repo with build artifacts.

### A.1 — `scripts/release.sh`

Bash script. Idempotent within reason — re-runs clean up prior staging dir before re-staging. Takes `<version>` arg; aborts cleanly with usage if missing.

```bash
#!/usr/bin/env bash
set -euo pipefail

#-----------------------------------------------------------------------
# scripts/release.sh — Build a Neato Hive release tarball.
#
# Usage:  ./scripts/release.sh <version>
#         e.g.  ./scripts/release.sh 1.5.0
#         e.g.  ./scripts/release.sh 1.5.0-rc.1
#
# Output:
#   /tmp/neato-hive-v<version>.tar.gz       (the release tarball)
#   /tmp/neato-hive-v<version>.checksums.txt (SHA-256 sidecar)
#
# Excludes node_modules/ per Q2 — update flow runs `pnpm install
# --frozen-lockfile` post-extract. ABI-safe across Node 18/20/22.
#
# Excludes everything in PRESERVE_LIST so an end-user's agents/, data/,
# config/*.local.yaml, .env files survive untouched across `hive update`.
#
# Does NOT push to site repo or update current.json — that's B.2.
#-----------------------------------------------------------------------

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
cd "${REPO_ROOT}"

#-- Argument validation --------------------------------------------------
if [ "$#" -ne 1 ]; then
  echo "Usage: $0 <version>"
  echo "  e.g. $0 1.5.0"
  echo "  e.g. $0 1.5.0-rc.1"
  exit 2
fi

VERSION="$1"
TARBALL="/tmp/neato-hive-v${VERSION}.tar.gz"
CHECKSUMS="/tmp/neato-hive-v${VERSION}.checksums.txt"
STAGING="${REPO_ROOT}/dist-pkg"

#-- Pre-flight checks ----------------------------------------------------
echo "==> Verifying clean repo state..."
if [ -n "$(git status --porcelain)" ]; then
  echo "==> ERROR: working tree has uncommitted changes. Commit or stash first."
  git status --short | head -10
  exit 1
fi

CURRENT_BRANCH=$(git rev-parse --abbrev-ref HEAD)
if [ "${CURRENT_BRANCH}" != "main" ]; then
  echo "==> WARNING: not on main (current: ${CURRENT_BRANCH}). Proceeding, but releases are normally cut from main."
fi

echo "==> Verifying package.json version matches '${VERSION}'..."
PKG_VERSION=$(node -e "console.log(require('./package.json').version)")
if [ "${PKG_VERSION}" != "${VERSION}" ]; then
  echo "==> ERROR: package.json version is '${PKG_VERSION}', expected '${VERSION}'."
  echo "==> Either bump package.json or pass the matching version."
  exit 1
fi

echo "==> Verifying packageManager field is present..."
PKG_MGR=$(node -e "console.log(require('./package.json').packageManager || '')")
if [ -z "${PKG_MGR}" ]; then
  echo "==> ERROR: package.json missing 'packageManager' field. Required for Q2 corepack pinning."
  exit 1
fi
echo "    packageManager = ${PKG_MGR}"

echo "==> Verifying CHANGELOG entry exists for v${VERSION}..."
if ! grep -qE "^## \[${VERSION}\]" CHANGELOG.md 2>/dev/null; then
  echo "==> ERROR: CHANGELOG.md missing entry for ## [${VERSION}]"
  echo "==> Add an entry then re-run."
  exit 1
fi

#-- Build phase ----------------------------------------------------------
echo "==> Running pnpm install --frozen-lockfile..."
pnpm install --frozen-lockfile

echo "==> Running pnpm build..."
pnpm build

echo "==> Running pnpm test..."
pnpm test

#-- Staging phase --------------------------------------------------------
echo "==> Cleaning prior staging dir..."
rm -rf "${STAGING}"
mkdir -p "${STAGING}"

echo "==> Staging REPLACE_LIST items into ${STAGING}/..."

# REPLACE_LIST — directories
for ITEM in dist bin templates shared skills dashboard; do
  if [ -d "${REPO_ROOT}/${ITEM}" ]; then
    echo "    + ${ITEM}/"
    cp -a "${REPO_ROOT}/${ITEM}" "${STAGING}/${ITEM}"
  else
    if [ "${ITEM}" = "dashboard" ]; then
      echo "    - ${ITEM}/ (skipped — Phase E placeholder, not present yet)"
    else
      echo "==> ERROR: required REPLACE_LIST directory '${ITEM}/' not found in repo root."
      exit 1
    fi
  fi
done

# REPLACE_LIST — top-level files
for FILE in package.json pnpm-lock.yaml; do
  if [ -f "${REPO_ROOT}/${FILE}" ]; then
    echo "    + ${FILE}"
    cp -a "${REPO_ROOT}/${FILE}" "${STAGING}/${FILE}"
  else
    echo "==> ERROR: required REPLACE_LIST file '${FILE}' not found in repo root."
    exit 1
  fi
done

# Generated VERSION file (NEW per-release artifact, not committed)
echo "    + VERSION (generated)"
echo -n "${VERSION}" > "${STAGING}/VERSION"

#-- Tarball -------------------------------------------------------------
echo "==> Creating tarball ${TARBALL}..."
rm -f "${TARBALL}"
tar -czf "${TARBALL}" -C "${REPO_ROOT}" -- dist-pkg/

#-- Checksum -----------------------------------------------------------
echo "==> Computing SHA-256..."
SHA=$(shasum -a 256 "${TARBALL}" | awk '{print $1}')
echo "${SHA}  $(basename "${TARBALL}")" > "${CHECKSUMS}"
echo "    SHA-256 = ${SHA}"

#-- Summary -----------------------------------------------------------
echo ""
echo "==> Release tarball ready:"
echo "    ${TARBALL}"
echo "    ${CHECKSUMS}"
echo ""
echo "==> Tarball size: $(du -h "${TARBALL}" | awk '{print $1}')"
echo "==> Inspect contents:  tar -tzf ${TARBALL} | head -30"
echo "==> Audit:             bash scripts/release-audit.sh ${VERSION}"
echo "==> B.2 will push to site repo + update current.json + index.json."
```

**Notes on shell discipline:**
- `set -euo pipefail` — fail fast on any error
- `cp -a` preserves permissions and symlinks
- `tar -czf ... -C "${REPO_ROOT}" -- dist-pkg/` — tar from the repo root, relative path, so extracted tarball produces `dist-pkg/` at extraction root (consumers extract under `~/neato-hive/.update-staging/<id>/` then atomic-overlay into `~/neato-hive/`, per C.2's spec)
- `shasum -a 256` is portable across macOS + Linux; `sha256sum` is Linux-only

### A.2 — `scripts/release-audit.sh`

Bash script. Compares the staged `dist-pkg/` contents against the expected REPLACE_LIST. Output is enumerated for human / CI inspection.

```bash
#!/usr/bin/env bash
set -euo pipefail

#-----------------------------------------------------------------------
# scripts/release-audit.sh — Audit the staged release contents.
#
# Reads dist-pkg/ produced by release.sh and prints:
#   - Items present that ARE in REPLACE_LIST (✓)
#   - Items present that are NOT in REPLACE_LIST (warning — possible
#     inadvertent leak from PRESERVE_LIST)
#   - REPLACE_LIST items NOT present (warning — possible missing
#     directory or skipped item)
#
# Usage:  ./scripts/release-audit.sh <version>  (version is informational only)
#-----------------------------------------------------------------------

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
STAGING="${REPO_ROOT}/dist-pkg"

if [ ! -d "${STAGING}" ]; then
  echo "==> ERROR: ${STAGING}/ not present. Run release.sh first."
  exit 1
fi

# Canonical REPLACE_LIST (must match release.sh + canonical v1.5.0 spec)
declare -a REPLACE_LIST=(
  "dist"
  "bin"
  "templates"
  "shared"
  "skills"
  "dashboard"        # Phase E placeholder — may be absent
  "package.json"
  "pnpm-lock.yaml"
  "VERSION"
)

# Items in PRESERVE_LIST (or build artifacts) that should NEVER appear
declare -a FORBIDDEN=(
  "node_modules"
  ".git"
  "src"
  "agents"
  "data"
  "config"
  ".env"
  ".env.local"
  "tsconfig.json"
  "pnpm-workspace.yaml"
)

echo "==> Audit of ${STAGING}/ contents:"
echo ""

# Enumerate what's in staging
mapfile -t ACTUAL < <(cd "${STAGING}" && ls -1A 2>/dev/null)

# Check expected items
echo "  REPLACE_LIST audit:"
for ITEM in "${REPLACE_LIST[@]}"; do
  if [ -e "${STAGING}/${ITEM}" ]; then
    echo "    ✓ ${ITEM}"
  else
    if [ "${ITEM}" = "dashboard" ]; then
      echo "    - ${ITEM} (skipped — Phase E placeholder, not present)"
    else
      echo "    ✗ ${ITEM}  (MISSING — release.sh did not stage it)"
    fi
  fi
done

# Check forbidden items
echo ""
echo "  PRESERVE_LIST sanity (these MUST be absent):"
LEAK=0
for ITEM in "${FORBIDDEN[@]}"; do
  if [ -e "${STAGING}/${ITEM}" ]; then
    echo "    ✗ ${ITEM}  (LEAKED INTO STAGING — fix release.sh)"
    LEAK=1
  fi
done
if [ "${LEAK}" -eq 0 ]; then
  echo "    ✓ no PRESERVE_LIST / forbidden items detected"
fi

# Detect anything unexpected
echo ""
echo "  Unexpected items in staging (outside REPLACE_LIST):"
UNEXPECTED=0
for ACT in "${ACTUAL[@]}"; do
  FOUND=0
  for EXPECTED in "${REPLACE_LIST[@]}"; do
    if [ "${ACT}" = "${EXPECTED}" ]; then
      FOUND=1
      break
    fi
  done
  if [ "${FOUND}" -eq 0 ]; then
    echo "    ? ${ACT}  (not in REPLACE_LIST — investigate)"
    UNEXPECTED=1
  fi
done
if [ "${UNEXPECTED}" -eq 0 ]; then
  echo "    ✓ no unexpected items"
fi

echo ""
if [ "${LEAK}" -eq 0 ] && [ "${UNEXPECTED}" -eq 0 ]; then
  echo "==> Audit clean. Tarball composition matches REPLACE_LIST."
  exit 0
else
  echo "==> Audit FAILED. See warnings above."
  exit 1
fi
```
